# Memory Ministry  
## Runtime / Ministries

---

## 1. Purpose

The Memory Ministry manages Edencore’s internal knowledge, context, and recall.  
It ensures continuity, stability, and coherence across interactions.

---

## 2. Responsibilities

- store relevant context  
- retrieve information when needed  
- maintain short‑term and long‑term memory  
- protect against drift or corruption  
- enforce Mindcore identity rules  

---

## 3. Boundaries

The Memory Ministry does not:

- interpret meaning (ESM)  
- execute actions (Runtime)  
- interact with devices (Edencore‑OS)  

---

## 4. Constitutional Statement

**Memory preserves continuity, identity, and clarity within Edencore.**
